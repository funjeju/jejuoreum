# 19. Oreum MBTI / "나를 닮은 오름"

> 본 문서는 오름 MBTI 시스템의 모든 명세를 정의한다.
> 검색 유입과 SNS 바이럴을 동시에 노리는 핵심 신규 기능이며,
> 패스포트 인지도가 없는 사용자에게 즉각적 가치를 제공하는 진입점이다.

---

## 0. 시스템 철학

### 0.1 왜 MBTI인가

**한국에서 MBTI는 압도적으로 강력한 검색 키워드**다. 이를 활용한 본 시스템은:

- **검색 유입의 새로운 입구**: "오름 MBTI", "{MBTI유형} 오름" 롱테일 SEO
- **SNS 바이럴 트리거**: 결과 공유가 자연 발생 ("나는 ENFJ — 따라비오름이래")
- **첫 방문자 정착률 상승**: 비로그인 사용자에게 즉각적 가치 제공
- **가입 동기 강화**: "이 결과 저장하려면 가입하세요"
- **데이터 자산화**: 사용자 성향 데이터 누적

### 0.2 본 시스템의 차별화

다른 MBTI 콘텐츠와 차이:

- **단순 결과 X**: 추천 오름이 함께
- **계속 사용 가치**: 실제로 그 오름에 다녀올 동기 제공
- **데이터 + 감성**: 오름 데이터 (좌표, 표고)와 감성 매칭의 결합
- **재테스트 동기**: 시간 지나면 다시 테스트, 결과 변할 수 있음

### 0.3 핵심 결정

- **10문항 + 보조 4문항** (총 14문항, 2분 이내)
- **MBTI 16유형 매핑은 기획자 직접 큐레이션**
- **결과 페이지 16종** (각 MBTI별 풀 콘텐츠)
- **비회원도 테스트 가능, 가입 시 결과 인계**
- **SNS 공유 카드 자동 생성**

## 1. MBTI 16유형 오름 매칭

### 1.1 16유형 그리드

| | T (사고형) | F (감정형) |
|---|---|---|
| **NJ (직관·계획)** | INTJ ENTJ | INFJ ENFJ |
| **NP (직관·인식)** | INTP ENTP | INFP ENFP |
| **SJ (감각·계획)** | ISTJ ESTJ | ISFJ ESFJ |

### 1.2 유형별 매칭 가이드 (예시)

각 MBTI를 오름 성격과 매칭하는 기준:

#### NT 그룹 (분석가)

**INTJ - 완벽주의 깊이파**
- 매칭 키워드: 깊은 분화구, 완벽한 형태, 학술적 가치
- 추천 오름 예: 다랑쉬오름 (오름의 여왕, 외형 완벽), 거문오름 (세계자연유산, 학술 가치)
- 경험 키워드: 혼자 깊이 음미, 정상에서의 사색

**INTP - 호기심 탐구자**
- 매칭 키워드: 독특한 지질, 비밀스러움, 사람 적음
- 추천 오름 예: 산굼부리 (분화구 명소), 영주산 (지질학적 흥미)
- 경험 키워드: 발견의 즐거움, 자신만의 해석

**ENTJ - 정복형 리더**
- 매칭 키워드: 도전, 정상의 광활함, 한라산 조망
- 추천 오름 예: 한라산 (제주 최고봉), 어승생악 (전망 일품)
- 경험 키워드: 정복감, 성취

**ENTP - 모험적 실험가**
- 매칭 키워드: 다양한 코스, 변화무쌍, 사진 명소
- 추천 오름 예: 새별오름 (대표적, 누구나 좋아함), 노꼬메오름 (다양한 풍경)
- 경험 키워드: 새로운 시도

#### NF 그룹 (외교관)

**INFJ - 깊은 신비주의자**
- 매칭 키워드: 신비로운 분위기, 영적, 일출/일몰
- 추천 오름 예: 영주산 (일출 명소), 따라비오름 (오름의 여왕, 영적 분위기)
- 경험 키워드: 영혼의 여행, 깊은 감동

**INFP - 감성적 시인**
- 매칭 키워드: 부드러운 능선, 감성적, 야생화
- 추천 오름 예: 용눈이오름 (부드러운 능선, 감성 명소), 백약이오름 (사진 명소)
- 경험 키워드: 시적인 풍경, 자기와의 대화

**ENFJ - 모두를 품는 리더**
- 매칭 키워드: 사방 풍경, 베스트, 누구나 좋아하는
- 추천 오름 예: 따라비오름 (오름의 여왕, 사방 풍경), 새별오름 (대중적 인기)
- 경험 키워드: 함께 나누는 감동

**ENFP - 자유로운 영혼**
- 매칭 키워드: 다채로운 풍경, 감성, 즉흥적
- 추천 오름 예: 새별오름 (다양한 매력), 백약이오름 (감성 충만)
- 경험 키워드: 자유, 영감

#### SJ 그룹 (관리자)

**ISTJ - 신뢰의 전통주의자**
- 매칭 키워드: 정통, 잘 정비된 코스, 안정감
- 추천 오름 예: 새별오름 (정비 잘됨), 산굼부리 (안정적 코스)
- 경험 키워드: 정직한 자연

**ISFJ - 따뜻한 보호자**
- 매칭 키워드: 안전, 온화, 가족 친화
- 추천 오름 예: 새별오름 (편안), 산굼부리 (산책로 좋음)
- 경험 키워드: 평화, 위로

**ESTJ - 체계적 실용주의자**
- 매칭 키워드: 정복 가능한, 명확한 코스, 효율
- 추천 오름 예: 따라비오름 (베스트), 다랑쉬오름 (대표적)
- 경험 키워드: 성취, 효율적 탐방

**ESFJ - 따뜻한 사교가**
- 매칭 키워드: 인기 명소, 함께 즐기는, 사진 잘 나오는
- 추천 오름 예: 새별오름 (대중 인기), 백약이오름 (사진 명소)
- 경험 키워드: 함께함, 인스타 감성

#### SP 그룹 (탐험가)

**ISTP - 조용한 장인**
- 매칭 키워드: 도전적, 사람 적음, 독립적
- 추천 오름 예: 한라산 윗세오름, 어승생악 (도전적)
- 경험 키워드: 자기만의 페이스

**ISFP - 감성적 예술가**
- 매칭 키워드: 부드러운 풍경, 사진, 감성
- 추천 오름 예: 용눈이오름 (감성 능선), 백약이오름 (예술적 풍경)
- 경험 키워드: 미적 감수성

**ESTP - 모험적 실용주의자**
- 매칭 키워드: 활동적, 도전, 즉흥
- 추천 오름 예: 한라산, 노꼬메오름 (도전적)
- 경험 키워드: 액션, 모험

**ESFP - 자유로운 엔터테이너**
- 매칭 키워드: 인기 명소, 화려한 풍경, 사진 잘 나오는
- 추천 오름 예: 새별오름 (대표 명소), 따라비오름 (사방 풍경)
- 경험 키워드: 즐거움, 화려함

### 1.3 매핑 작업 방식

기획자가 360개 오름을 직접 매핑.

각 오름에 대해:
- **메인 MBTI 1개**: 가장 강하게 매칭되는 유형
- **보조 MBTI 1~3개**: 부분적으로 매칭되는 유형들
- **MBTI traits**: `{energy: 'extrovert', sensing: 'intuition', ...}` 형식 (선택)

```sql
-- 예시 매핑
UPDATE oreums SET
  mbti = 'ENFJ',
  mbti_secondary = ARRAY['ESFJ', 'INFJ', 'ENFP'],
  mbti_traits = '{
    "main_traits": ["여왕", "포용", "사방조망"],
    "energy_level": 7,
    "social_nature": 8
  }'::jsonb
WHERE slug = 'darangshi';
```

### 1.4 매핑 균형

16개 MBTI에 360개 오름이 균형 있게 분포해야 함:
- 평균: 한 유형당 약 22개 오름
- 100선 기준: 한 유형당 약 6~7개

**매핑 검증**:
- 한 유형에 너무 적거나 (3개 미만) 너무 많은 (50개 이상) 오름이 매핑되면 재조정
- 100선 우선 매핑 검증 (각 유형에 100선 5개+ 보장)

## 3. 매칭 알고리즘

### 3.1 단계별 흐름

```
[사용자 답변 (10 + 4문항)]
    ↓
[1단계: MBTI 결정]
- 각 차원별 점수 합산
- E/I, S/N, T/F, J/P 각 결정
    ↓
[2단계: 1차 매칭]
- mbti = 결정된 유형 + 100선 우선
    ↓
[3단계: 보조 변수로 좁히기]
- 선호 지역, 난이도, 계절, 시간대 적용
    ↓
[4단계: 최종 1~3개 추천]
- 1순위: 가장 매칭 점수 높은 1개
- 2~3순위: 보조 추천
    ↓
[5단계: 결과 캐시 + 로그]
- user_quiz_results.recommended_oreum_ids
```

### 3.2 MBTI 결정 로직

```typescript
function determineMbti(answers: Answer[]): {
  mbti: string;
  scores: { EI: {E: number; I: number}; SN: {...}; TF: {...}; JP: {...} };
} {
  const scores = {
    EI: { E: 0, I: 0 },
    SN: { S: 0, N: 0 },
    TF: { T: 0, F: 0 },
    JP: { J: 0, P: 0 }
  };

  for (const ans of answers) {
    const question = await getQuestion(ans.question_id);
    const dim = question.dimension; // 'EI' | 'SN' | 'TF' | 'JP'
    const value = ans.selected === 'A' ? question.option_a_value : question.option_b_value;
    scores[dim][value]++;
  }

  // 각 차원별 우세 결정
  const EI = scores.EI.E >= scores.EI.I ? 'E' : 'I';
  const SN = scores.SN.S >= scores.SN.N ? 'S' : 'N';
  const TF = scores.TF.T >= scores.TF.F ? 'T' : 'F';
  const JP = scores.JP.J >= scores.JP.P ? 'J' : 'P';

  return {
    mbti: EI + SN + TF + JP, // 예: 'ENFJ'
    scores
  };
}
```

### 3.3 보조 변수 매칭

```sql
-- 1차 매칭: 정확한 MBTI + 100선
WITH primary_matches AS (
  SELECT *, 1 AS match_priority,
    -- 보조 변수 매칭 점수 (0~4)
    (CASE WHEN region = $preferred_region OR $preferred_region = 'any' THEN 1 ELSE 0 END +
     CASE WHEN difficulty = $preferred_difficulty OR $preferred_difficulty IS NULL THEN 1 ELSE 0 END +
     CASE WHEN $preferred_season = ANY(recommended_seasons) OR $preferred_season = 'any' THEN 1 ELSE 0 END +
     CASE WHEN $preferred_time = ANY(recommended_times) OR $preferred_time = 'any' THEN 1 ELSE 0 END
    ) AS bonus_score
  FROM oreums
  WHERE mbti = $user_mbti
    AND is_top_100 = true
    AND is_published = true
),
-- 2차: 보조 MBTI + 100선
secondary_matches AS (
  SELECT *, 2 AS match_priority,
    (CASE WHEN region = $preferred_region OR $preferred_region = 'any' THEN 1 ELSE 0 END +
     ...) AS bonus_score
  FROM oreums
  WHERE $user_mbti = ANY(mbti_secondary)
    AND is_top_100 = true
    AND is_published = true
    AND id NOT IN (SELECT id FROM primary_matches)
),
-- 3차: 정확 MBTI + 100선 외
fallback_matches AS (
  SELECT *, 3 AS match_priority,
    (...) AS bonus_score
  FROM oreums
  WHERE mbti = $user_mbti
    AND is_top_100 = false
    AND is_published = true
)
SELECT * FROM (
  SELECT * FROM primary_matches
  UNION ALL
  SELECT * FROM secondary_matches
  UNION ALL
  SELECT * FROM fallback_matches
) all_matches
ORDER BY
  match_priority ASC,
  bonus_score DESC,
  RANDOM() -- 동점 시 랜덤 (재테스트 시 다양성)
LIMIT 3;
```

### 3.4 매칭 결과 다양성

같은 사용자가 여러 번 테스트하면:
- 1순위는 가장 강한 매칭 (변동 적음)
- 2~3순위는 ORDER BY RANDOM()으로 다양성 부여

이게 **재테스트 동기**가 됨 ("두 번째 결과는 뭐가 나올까?").

### 3.5 한계 처리

매칭 결과 0개:
- 추가 fallback: 비100선 보조 MBTI도 검색
- 그래도 0개면: 같은 지역 추천
- 최종 fallback: 100선 랜덤 1개

### 13.2 360개 MBTI 매핑 가이드

기획자가 효율적으로 매핑하려면:

1. **시드 매핑** (1주차):
   - 16 MBTI별 대표 오름 5~10개 선정
   - 명확한 매칭부터 시작

2. **확장 매핑** (2주차):
   - 나머지 오름들을 비슷한 패턴으로 분류
   - 16 MBTI 그리드를 채워가는 방식

3. **검증** (3주차):
   - 한 유형에 너무 많거나 적은지 확인
   - 100선 우선 검증
   - 균형 조정

도구:
- 백오피스의 MBTI 매핑 도구 (`14_admin_backoffice.md` 6.4)

### 13.3 16개 결과 페이지 콘텐츠

각 페이지당 작성 분량:
- 유형 설명 (50~100자)
- 왜 닮았는지 서사 (200~300자)
- 유형 특성 키워드 (3~5개)
- SNS 공유 카피 (50자 이내)
- 다른 MBTI와의 연결 (100~200자)

총 16개 = 약 1만 1천 자.

권장 작성 방식:
- 기획자 직접 작성 (서사가 핵심, AI 보조 한계)
- AI로 1차 드래프트 → 기획자 풀 보강
