# 05. Oreum Card Page

> 본 문서는 개별 오름의 디지털 카드 페이지(`/oreum/{slug}`) 상세 명세를 정의한다.
> 사용자가 인증 후 가장 자주 들어오는 화면 중 하나이며, SEO 콘텐츠 페이지와 통합된 구조를 갖는다.

---

## 0. 페이지의 역할

### 0.1 두 종류의 사용자 진입

본 페이지는 **두 가지 다른 진입 시나리오**를 모두 만족해야 한다.

**진입 A: 도감 사용자**
- 마이페이지 도감에서 카드 클릭
- 자기가 발견한 오름 카드 다시 보기
- 메모, 사진, 후기 추가
- 위시리스트 추가 (미발견 시)

**진입 B: SEO 검색 유입**
- 구글/네이버에서 "다랑쉬오름" 검색
- 비로그인 상태로 진입
- 정보 탐색
- 가입·패스포트 흥미

이 두 시나리오를 한 페이지가 담는다. 단, 사용자 상태에 따라 노출이 달라진다.

### 0.2 통합 설계 원칙

- **로그인 없이도 모든 정보 열람 가능** (SEO 정책)
- **로그인 시 개인화 영역 추가** (내 기록, 카드 상태)
- **SEO 페이지와 도감 카드 페이지는 같은 URL** (`/oreum/{slug}`)
- 페이지 내부에서 사용자 상태로 분기

### 0.3 핵심 디자인 원칙

- **카드 비주얼 sticky**: 어느 섹션을 보든 어떤 오름인지 시각적 컨텍스트 유지
- **레이어 메뉴**: 카드 위에 떠있는 반투명 메뉴로 빠른 섹션 이동
- **스크롤 한 번으로 모든 정보**: 탭 전환 X, 부드러운 스크롤로 통합
- **발견 상태별 다른 CTA**: 미발견은 "위시리스트 추가", 발견은 "기록 추가"

```
┌────────────────────────────────────┐

├────────────────────────────────────┤
│                                    │
│  [Breadcrumb]                       │ ← SEO용
│  홈 > 동부 > 다랑쉬오름                │
│                                    │
│  [카드 비주얼 영역]                   │ ← 380px (sticky from top)
│  - 카드 일러스트 (full)              │
│  - 오름 이름 (오버레이)               │
│  - 핵심 메타 (오버레이)               │
│  - 발견 상태 마크                    │
│                                    │
│  [레이어 메뉴 (카드 위에 sticky)]      │ ← 48px
│  [정보][등반][주변][갤러리][후기][내기록]│

│  (이름의 유래, 형성 배경)            │
│                                    │
│  ── 섹션 3: 지질·생태 ──            │
│                                    │
│  ── 섹션 4: 탐방 정보 ──            │
│  (코스, 시간, 난이도)                │
│                                    │
│  ── 섹션 5: 가는 길 ──              │
│  (주차장, 대중교통, 지도 임베드)      │
│                                    │
│  ── 섹션 6: 베스트 시즌 ──          │
│                                    │
│  ── 섹션 7: 포토 포인트와 갤러리 ──   │
│                                    │

│                                    │

│                                    │
│  ── 섹션 10: 내 기록 ── (로그인+발견)│

│  [하단 고정 CTA]                     │ ← 64px (sticky bottom)

│  스트 추가"                          │
└────────────────────────────────────┘
        ↓ 하단 탭바 (로그인 사용자만)
```

### 1.2 sticky 요소 설명

상단부터:
1. **앱 헤더** (56px): 항상 sticky
2. **카드 비주얼** (380px): 화면 상단에 도달하면 80px로 축소되어 sticky
3. **레이어 메뉴** (48px): 카드 비주얼 아래에 항상 sticky

스크롤 시 시각 변화:
- 카드 비주얼이 점점 작아짐 (380 → 80px)
- 카드 비주얼이 작아질 때 오름 이름이 헤더로 이동
- 레이어 메뉴는 항상 같은 위치

이 동작은 iOS 인스타그램 프로필 스크롤이나 iOS 노트 앱과 비슷한 패턴.

## 3. 카드 비주얼 영역 (Hero)

### 3.1 기본 상태 (스크롤 전)

```
높이: 380px
배경: 카드 일러스트 (full bleed)
오버레이: 사진 위 다크 그라데이션 (하단 30%)
모서리: 0 (페이지 가장자리까지)

콘텐츠 위치:

좌상단 (44px from top):
- Breadcrumb (작은 글씨)
- 12px, white opacity 0.85

좌하단:
- 카드 번호: "001 / 비기너"
  · 12px, white opacity 0.85
- 오름 이름: "다랑쉬오름"
  · 28px, weight 700, white
- 영문 이름: "Darangshi Oreum"
  · 14px, weight 400, white opacity 0.85
- 한 줄 요약: "제주 동부의 여왕"
  · 14px, weight 500, white

우하단:
- 핵심 메타 (작은 카드 형태):
  ┌──────────┐
  │ ⛰️ 382m  │
  │ ⏱️ 60분  │
  │ ★★★☆☆  │
  └──────────┘
  · 반투명 white 배경
  · 백드롭 블러
  · 11px, white

우상단 (52px from top):
- 발견 상태:
  발견: 원형 44px, --brand-green-800 + 흰 체크 + 발견일
  미발견: 점선 원형 44px, white 30%
```

### 3.2 sticky 축소 상태

스크롤이 카드 영역을 지났을 때:

```
높이: 80px (축소됨)
배경: 카드 일러스트 (어둡게 처리)
오버레이: 더 진한 다크

콘텐츠:
- 오름 이름이 헤더 영역으로 이동
- 핵심 메타는 숨김
- 발견 상태 마크만 작게 우상단 (28px)
```

### 3.3 100선 외 페이지의 경우

비100선 오름 SEO 페이지는 카드 비주얼이 다름:
- 일러스트 X (100선만 일러스트 보유)
- 사진 또는 단색 배경
- "100선 외" 라벨 표시
- 카드 위에 "100선이 아닌 오름이에요" 알림

### 3.4 발견 모먼트와의 연결

도감에서 카드 탭 시 Hero animation으로 부드럽게 진입.

```
도감 탭 카드 클릭 시:
[카드 일러스트가 확대되면서 페이지 전환]
   - duration: 350ms
   - 카드 위치에서 시작 → Hero 영역에 안착
   - 다른 콘텐츠는 fade-in
```

## 4. 레이어 메뉴

### 4.1 구조

```
┌────────────────────────────────────┐
│ [한눈에] [등반] [주변] [갤러리] [후기]│
└────────────────────────────────────┘

높이: 48px
배경: rgba(255, 255, 255, 0.85)
백드롭 블러: blur(12px)
보더 하단: 0.5px solid --neutral-200
```

### 4.2 메뉴 항목

기본 5개:
1. 한눈에 보기 (섹션 1)
2. 등반 (섹션 4)
3. 주변 (섹션 8)
4. 갤러리 (섹션 7)
5. 후기 (섹션 9)

발견한 사용자에게 추가:
6. 내 기록 (섹션 10) — 활성 시 가장 좌측 표시

### 4.3 사양

```
각 항목:
- 패딩: 8px 12px
- 폰트: 13px, weight 500
- 색상 (비활성): --neutral-700
- 색상 (현재 보고 있는 섹션): --brand-green-800
- 활성 표시: 하단 보더 2px solid --brand-green-800

가로 스크롤:
- 화면 작거나 항목 많을 시 가로 스크롤 가능
- 활성 항목이 화면 가운데로 자동 스크롤
```

### 4.4 클릭 동작

```
[메뉴 항목 클릭]
    ↓
[해당 섹션으로 부드러운 스크롤]
- duration: 600ms
- easing: cubic-bezier(0.4, 0, 0.2, 1)
- offset: sticky 헤더 + 메뉴 높이만큼 보정
    ↓
[활성 표시 갱신]
```

### 4.5 스크롤 시 자동 활성화

```
[사용자가 스크롤]
    ↓
[Intersection Observer로 현재 섹션 감지]
    ↓
[해당 메뉴 항목 자동 활성화]
    ↓
[가로 스크롤 메뉴에서 해당 항목이 가운데로]
```

## 6. 섹션 2: 오름 소개

### 6.1 구조

```
배경: --white
패딩: --space-6

헤딩: "오름 소개" (h2, 18px, weight 600)

본문:
- 이름의 유래 (1~2문단)
- 형성 배경 (1~2문단)
- 역사·문화적 의미 (1~2문단)

폰트:
- 본문: 15px, line-height 1.7
- 색상: --neutral-900

분량: 250~350자 (SEO 표준)
```

### 6.2 콘텐츠 출처

`seo_contents` 테이블의 `content_sections` JSONB에서 가져옴.

```typescript
interface ContentSection {
  section: 'introduction';
  content: string; // markdown 또는 plain text
}
```

### 6.3 다국어 처리

```typescript
// 사용자 언어에 따라 콘텐츠 fetch
const lang = userSettings?.preferred_language || 'ko';
const content = await getSeoContent(oreum.slug, lang);

// 해당 언어 콘텐츠 부재 시 fallback
if (!content) {
  // /ko로 자동 리다이렉트 (SEO 페널티 회피)
  redirect(`/ko/oreum/${slug}`);
}
```

### 19.1 sticky 카드 축소

```javascript
useEffect(() => {
  const handleScroll = () => {
    const scrollY = window.scrollY;
    const threshold = 380; // 카드 영역 높이

    if (scrollY > threshold) {
      // 카드 축소 모드
      setHeroState('compact');
    } else {
      // 카드 확장 모드
      const progress = scrollY / threshold;
      setHeroProgress(progress); // 0~1
    }
  };

  window.addEventListener('scroll', handleScroll, { passive: true });
  return () => window.removeEventListener('scroll', handleScroll);
}, []);
```

// Intersection Observer로 현재 보이는 섹션 감지
useEffect(() => {
