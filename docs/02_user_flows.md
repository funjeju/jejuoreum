# 02. User Flows

> 본 문서는 제주 오름 패스포트 프로젝트의 모든 사용자 시나리오를 정의한다.
> 각 흐름은 트리거, 상태 전환, 분기 조건, 결과를 명시한다.
> 본 문서는 `core.md v0.4`의 5가지 신규 기능 결정(오름 MBTI, 리듬, 다국어, 팀 미도입)을 반영한다.

---

## 0. 사용자 페르소나

설계의 명확성을 위해 핵심 페르소나 4종을 정의한다.

### 0.1 페르소나 A: 검색 유입 신규 사용자

- **유입 경로**: 구글/네이버에서 "다랑쉬오름", "제주 오름 추천", "오름 MBTI" 등 검색
- **현재 상태**: 패스포트 존재를 모름
- **목적**: 정보 탐색 (어디 갈지 결정 중)
- **이상적 결과**: 콘텐츠로 가치 인식 → MBTI 퀴즈 또는 가입 → 패스포트 흥미

### 0.2 페르소나 B: 패스포트 보유 활성 사용자

- **유입 경로**: 인쇄물 패스포트 구매 후 QR 스캔
- **현재 상태**: 가입 완료, 도감 채우는 중
- **목적**: 다음 갈 오름 결정, 인증, 기록
- **이상적 결과**: 만족스러운 인증 경험 + 재방문

### 0.3 페르소나 C: 외국인 관광객

- **유입 경로**: Google에서 "Jeju Oreum hike" 등 영어 검색 (페이즈 2 이후)
- **현재 상태**: 한국어 모름
- **목적**: 짧은 제주 여행 중 가볼 오름 결정
- **이상적 결과**: 영어 콘텐츠 충실 → 가벼운 사용

### 0.4 페르소나 D: 마니아 사용자

- **유입 경로**: 이미 100선 다 모은 사용자
- **현재 상태**: 발견률 100%, 챌린지·기여 위주 활동
- **목적**: 새로운 도전, 사진 기여, 마스터 영역 정복
- **이상적 결과**: 마스터 영역, 시즌 한정 챌린지 등 깊이 있는 콘텐츠

## 1. 진입 흐름 (Entry Flows)

### 1.1 검색 엔진 → SEO 페이지 진입

```
[Google/Naver 검색]
    ↓
[검색 결과 클릭]
    ↓
[/{lang}/oreum/{slug} 진입]
    ↓
[언어 감지: Accept-Language]
    ↓
─── 한국어 사용자 → /ko/oreum/{slug}
─── 영어 사용자 → /en/oreum/{slug} (페이즈 2 이후)
─── 그 외 → /ko/oreum/{slug} (기본값)
    ↓
[SEO 페이지 표시]
    ↓
[하단 CTA 노출]
   ├── "이 오름을 도감에 추가" (회원가입 유도)
   ├── "오름 MBTI 테스트" (참여 유도)
   └── "패스포트 구매" (인쇄물 안내)
```

**중요**: SEO 페이지는 **로그인 없이 모든 콘텐츠 열람 가능**. 가입 마찰 최소화.

### 1.2 QR 스캔 → 진입

QR이 인쇄물 표지 안쪽 또는 뒷면에 단일 인쇄됨.

```
[QR 스캔]
    ↓
[/qr 진입 (단일 진입점)]
    ↓
[GPS 권한 요청]
    ↓
─── 권한 허용 ──┐
─── 권한 거부 → [/select 위시리스트에서 직접 선택]
                ↓
[현재 위치 측정]
    ↓
[GPS 3단계 매칭]
   ├── < 300m → [자동 매칭, 인증 진행]
   ├── 300m ~ 1km → [후보 2~3개 제시]
   └── ≥ 1km → [메인 화면으로 안내]
```

상세 인증 로직: `03_qr_gps_auth.md` 참조.

### 1.3 직접 URL 진입 (재방문)

PWA 홈 화면 추가한 사용자 또는 즐겨찾기 사용자.

```
[홈 화면 PWA 아이콘 탭 또는 즐겨찾기]
    ↓
[/ 메인 진입]
    ↓
─── 로그인 상태 → [홈 (탐험 시작형 또는 도감 중심형)]
─── 비로그인 → [공개 메인 (가입 유도)]
```

### 1.4 SNS 공유 링크 진입

타인의 MBTI 결과 공유 또는 코스 공유 링크.

```
[SNS에서 공유 카드 탭]
    ↓
[해당 콘텐츠 페이지 직접 진입]
   ├── /quiz/result/{mbti} (MBTI 결과)
   ├── /course/{share_token} (코스 공유)
   └── /oreum/{slug} (개별 오름)
    ↓
[비로그인 상태에서 콘텐츠 노출]
    ↓
[하단 CTA: "나도 해보기" / "도감에 추가" / "패스포트 알아보기"]
```

## 2. MBTI 퀴즈 흐름

### 2.1 퀴즈 진입

```
[메인 / SEO 페이지 / 광고 등]
    ↓
[/quiz 진입]
    ↓
[퀴즈 인트로 화면]
   - "10문항으로 알아보는 나를 닮은 오름"
   - 예상 소요 시간 (약 2분)
   - "시작하기" 버튼
    ↓
[퀴즈 진행 (10문항)]
```

### 2.2 퀴즈 진행 흐름

```
[1문항] (E/I 차원, 예: "정상에 도착하면?")
   ├── 선택 A
   └── 선택 B
    ↓
[2문항] (E/I 차원)
    ↓
... (10문항)
    ↓
[보조 질문 4개]
   - 선호 지역 (동/서/남/북/중산간/상관없음)
   - 선호 난이도 (1~5/상관없음)
   - 선호 계절 (봄/여름/가을/겨울/상관없음)
   - 선호 시간대 (일출/오전/오후/일몰/야간/상관없음)
    ↓
[로딩 (1~2초, 분석 애니메이션)]
    ↓
[/quiz/result/{mbti} 결과 페이지]
```

### 2.3 결과 페이지 흐름

```
[결과 페이지 진입]
    ↓
[애니메이션: 발견 모먼트 (흑백→컬러)]
   - 추천 오름 카드가 점차 컬러로 채워짐
    ↓
[결과 표시]
   ├── "당신은 ENFJ — ○○오름과 닮았습니다"
   ├── "왜 닮았는지" 서사 (200~300자)
   ├── 추천 오름 카드 (1순위 큰 카드)
   ├── 보조 추천 1~2개
   ├── 추천 오름의 핵심 정보
   └── 액션 버튼들
       ├── "이 오름 도감에 추가" → 회원가입 흐름
       ├── "오름 자세히 보기" → /oreum/{slug}
       ├── "결과 공유하기" → SNS 공유 카드
       └── "다시 테스트하기" → /quiz
```

### 2.4 비회원 → 회원 전환 흐름

```
[결과 페이지에서 "도감에 추가" 클릭]
    ↓
[가입 유도 모달]
   - "결과를 저장하고 도감을 시작해보세요"
   - 카카오/네이버 로그인 버튼
    ↓
[OAuth 인증]
    ↓
[가입 완료]
    ↓
[비회원 시 저장된 session_id로 user_quiz_results 인계]
   - user_id 채워줌
   - 추천 오름 자동으로 위시리스트에 추가
    ↓
[홈 화면 진입, 환영 메시지]
   - "○○오름이 위시리스트에 추가됐어요"
```

### 2.5 SNS 공유 카드 자동 생성

```
[결과 페이지에서 "공유" 클릭]
    ↓
[공유 카드 자동 생성]
   - 1080x1080 정사각 이미지
   - "나는 ENFJ — 닮은 오름은 ○○오름이래!"
   - 추천 오름 카드 일러스트 + 핵심 정보
   - 우리 사이트 URL 워터마크
    ↓
[공유 채널 선택]
   ├── 카카오톡
   ├── 인스타그램 스토리
   ├── 트위터
   ├── 페이스북
   └── 링크 복사
```

## 3. 회원가입 흐름

### 3.1 표준 가입 흐름

```
[가입 트리거]
   - 인증 시도 (QR 스캔 후)
   - 도감 추가 시도
   - MBTI 결과 저장
   - 사진 업로드 시도
    ↓
[가입 모달 또는 페이지]
   - 카카오 로그인 (한국 사용자 우선)
   - 네이버 로그인
   - (페이즈 2+) 구글 로그인
    ↓
[OAuth 인증]
    ↓
[필수 동의]
   - 서비스 이용약관
   - 개인정보 처리방침
   - 위치 정보 수집 동의 (선택)
   - 푸시 알림 (선택)
    ↓
[닉네임 설정]
   - OAuth에서 가져온 이름 기본값
   - 중복 체크
   - 변경 가능
    ↓
[온보딩]
   ├── (옵션) MBTI 결과 인계 알림
   ├── (옵션) 거주지 입력 (출발지 기본값용, 선택)
   ├── (옵션) 다국어 설정
   └── 환영 메시지 + 첫 추천
    ↓
[홈 화면]
```

### 3.2 비회원 데이터 인계

비회원 상태에서 한 활동을 가입 후 보존.

```
[비회원 활동 발생]
   - MBTI 퀴즈 완료
   - 위시리스트 추가
   - 페이지 즐겨찾기
    ↓
[로컬 스토리지 + session_id로 저장]
    ↓
[가입 시점]
    ↓
[session_id 매칭하여 데이터 인계]
   - user_quiz_results.user_id 채우기
   - 임시 위시리스트를 user_wishlist 테이블로 마이그레이션
    ↓
[가입 완료 후 환영 화면에서 인계 결과 노출]
   - "이전에 한 ○○ 활동을 이어가실 수 있어요"
```

### 3.3 가입 거부·이탈 처리

```
[가입 모달에서 닫기 또는 뒤로가기]
    ↓
[비회원 모드 유지]
    ↓
[다음 가입 트리거 시점에 다시 노출]
   - 단, 같은 세션에서 5회 이상 거부 시 24시간 안내 X
   - 푸시 노출 X
```

## 4. 인증 흐름 (오름 발견)

### 4.1 메인 인증 흐름 (GPS 자동 매칭)

```
[QR 스캔 또는 홈에서 "GPS 인증 시작"]
    ↓
[GPS 권한 확인]
   ├── 미허용 → [권한 요청 모달]
   └── 허용됨 → 다음
    ↓
[현재 위치 측정 (3~5초)]
    ↓
[가까운 100선 오름 검색]
    ↓
[3단계 매칭]
   ├── 가장 가까운 오름이 < 300m
   │     → 자동 매칭, "○○오름에 도착했어요!"
   ├── 300m ~ 1km
   │     → "근처 오름 후보 2~3개 중 선택하세요"
   └── ≥ 1km
         → "주변에 오름이 없어요. 위시리스트에서 선택하시겠어요?"
    ↓
[발견 모먼트 트리거]
   - 카드 흑백→컬러 전환 애니메이션
   - 진척도 카운터 업데이트
   - 햅틱 피드백
   - 신규 배지 획득 시 추가 모먼트
    ↓
[발견 후 액션]
   ├── 메모 입력 (선택)
   ├── 사진 업로드 (선택)
   └── 닫기 → 마이페이지로
```

### 4.2 분기: 자동 매칭 (< 300m)

```
[자동 매칭]
    ↓
[확인 화면]
   - 오름 이름과 사진
   - "○○오름에 도착했어요"
   - 인증 시점 날씨 자동 캡처
   - "맞다면 인증" / "다른 오름인데" 버튼
    ↓
─── "맞다면 인증" → 발견 모먼트
─── "다른 오름인데" → 후보 선택 화면
```

### 4.3 분기: 후보 제시 (300m ~ 1km)

```
[후보 제시]
    ↓
["근처에 다음 오름들이 있어요"]
   - 후보 2~3개 카드 (거리·이름·일러스트)
   - 거리순 정렬
   - "다녀온 오름은?" 안내
    ↓
─── 사용자 선택 → 발견 모먼트
─── "위에 없어요" → [/select 전체 위시리스트]
```

### 4.4 분기: 오름 없음 (≥ 1km)

```
[오름 없음 화면]
    ↓
["주변 1km 안에 오름이 없어요"]
   - 다음 옵션 제공:
   ├── "위시리스트에서 선택" (이미 다녀온 오름)
   ├── "지도에서 가장 가까운 오름 보기"
   └── "메인으로 돌아가기"
```

### 4.5 발견 모먼트 시퀀스

```
[발견 확정]
    ↓
[t=0ms] 화면 전환 시작
    ↓
[t=0~200ms] 카드 살짝 확대 (scale 1.0 → 1.05)
    ↓
[t=0~600ms] 흑백→컬러 전환 (saturation 0% → 100%)
    ↓
[t=300~600ms] 우상단 체크 마크 페이드인 + 튕김
    ↓
[t=300~1100ms] 진척도 숫자 카운트 애니메이션 (예: 36→37)
    ↓
[t=400ms] 햅틱 피드백 1회
    ↓
[t=600~1200ms] 카드 원래 크기 (scale 1.05 → 1.0)
    ↓
[t=1200ms] 신규 배지 체크
   ├── 신규 배지 획득 시 → 배지 모달 노출
   └── 없으면 → 다음 액션 옵션 표시
    ↓
[다음 액션 화면]
   - "메모 추가하기" / "사진 추가하기" / "다음으로"
```

상세 디자인 명세: `16_design_system.md`의 8.1 발견 모먼트 참조.

## 5. 마이페이지·도감 흐름

### 5.1 메인화면 진입

본 프로젝트는 메인화면 A(탐험 시작형)와 B(도감 중심형) 두 안을 시안으로 받았다.
**최종 결정**: 두 화면을 **별도 탭**으로 분리. (`16_design_system.md` 부록 참조)

- 홈 탭 (`/home`) → 메인화면 A 톤 (오늘의 추천, GPS 인증, 최근 발견)
- 도감 탭 (`/collection`) → 메인화면 B 톤 (그리드 위주, 진척도 시각화)

```
[로그인 사용자 진입]
    ↓
[기본값: 홈 탭]
    ↓
─── 홈 탭 → 활동 흐름 중심
─── 도감 탭 → 수집 시각화 중심
─── 위시리스트 탭
─── 마이 탭 (배지·설정·MBTI 결과)
```

### 5.2 도감 그리드 흐름

```
[도감 탭 진입]
    ↓
[상단: 발견률 시각화]
   - "36 / 100"
   - 진척도 바
   - "이번 달 리듬" 카드 (5.4 참조)
    ↓
[필터 영역]
   - 난이도: 비기너 / 익스플로러 / 마스터
   - 지역: 동 / 서 / 남 / 북 / 중산간
   - 발견 상태: 전체 / 발견 / 미발견
   - 정렬: 발견순 / 번호순 / 이름순
    ↓
[그리드 표시]
   - 100개 오름 카드 (3열)
   - 발견: 풀컬러
   - 미발견: 흑백 실루엣
    ↓
```

## 16. 성능·UX 고려사항

### 16.1 로딩 우선순위

- **Critical**: 발견 모먼트는 절대 로딩 지연 X (낙관적 업데이트)
- **High**: 카드 일러스트, 본인 데이터
- **Medium**: 갤러리 사진, 후기
- **Low**: 통계 view, 협력감 표시

### 16.2 오프라인 지원

- 마이페이지: 마지막 데이터 캐시
- 카드 페이지: stale-while-revalidate
- 인증: 오프라인 큐 + 백그라운드 동기화
- 사진 업로드: 오프라인 큐

### 16.3 푸시 알림 정책

**최소화 원칙** — 사용자를 귀찮게 하지 않음.

발송 트리거:
- 챌린지 완료
- 신규 배지 획득
- 친구 도감 비교 (페이즈 3)
- 시즌 새 챌린지 등장

발송 안 함:
- 일반 인증
- 사진 업로드 완료
- 일상적 활동
