# 14. Admin Backoffice

> 본 문서는 운영자가 사용하는 모든 백오피스 도구의 명세를 정의한다.
> 콘텐츠 큐레이션, 사용자 관리, 챌린지 운영, 제휴 상권 관리, SEO 콘텐츠 편집 등
> 운영 효율의 핵심 도구다.

---

## 0. 시스템 철학

### 0.1 백오피스의 역할

본 프로젝트는 **운영자의 큐레이션이 핵심 가치**다. 단순 자동화로는 만들 수 없는 부분:

- 사진의 카드 페이지 대표성 판단
- 코멘트의 가치 평가 (현장 팁 승격)
- SEO 콘텐츠의 깊이 있는 보강
- 챌린지의 시즌·이슈 큐레이션
- 제휴 상권의 적합성 검토

이 모든 작업이 **운영자 1~2명이 효율적으로 처리할 수 있어야** 한다.

### 0.2 핵심 설계 원칙

1. **모든 도구를 한 곳에**: 분산된 도구 X, 통합 백오피스
2. **빠른 처리**: 키보드 단축키, Tinder 스와이프, 일괄 액션
3. **모바일 친화**: 운영자도 이동 중 처리 (특히 사진 큐레이션)
4. **감사 로그**: 모든 운영 액션이 audit_logs에 기록
5. **권한 분리**: super_admin / editor / curator / merchant_manager

### 0.3 핵심 결정

- **별도 도메인**: `admin.jejuoreum.com`
- **로그인 강화**: 2FA 필수 (페이즈 2)
- **모바일 우선 일부**: 사진 큐레이션은 모바일에서도 사용 가능
- **데스크탑 우선**: SEO 편집기 등은 데스크탑만

## 1. 백오피스 진입·인증

### 1.1 접근 URL

```
https://admin.jejuoreum.com
```

별도 서브도메인. 일반 사용자에게 노출 안 됨.

### 1.2 로그인 흐름

```
[admin.jejuoreum.com 진입]
    ↓
[로그인 폼]
- 이메일
- 비밀번호
- (페이즈 2) 2FA 코드
    ↓
[admin_users 테이블 검증]
- user_id 매칭
- role 확인
    ↓
[role 기반 권한 체크 후 진입]
    ↓
[메인 대시보드]
```

### 1.3 권한 구조

| Role | 권한 |
|------|------|
| `super_admin` | 모든 권한 + 다른 admin 관리 |
| `editor` | SEO 콘텐츠 편집 + 오름 데이터 관리 |
| `curator` | 사진·코멘트 큐레이션 + 챌린지 생성 |
| `merchant_manager` | 제휴 상권 관리 |

권한 매트릭스:

| 기능 | super | editor | curator | merchant |
|------|-------|--------|---------|----------|
| 사진 승인 | ✓ | - | ✓ | - |
| 코멘트 승격 | ✓ | - | ✓ | - |
| 오름 마스터 수정 | ✓ | ✓ | - | - |
| SEO 편집 | ✓ | ✓ | - | - |
| 챌린지 생성 | ✓ | ✓ | ✓ | - |
| 제휴 상권 관리 | ✓ | - | - | ✓ |
| 사용자 관리 | ✓ | - | - | - |
| admin 추가/삭제 | ✓ | - | - | - |

### 1.4 세션·보안

- 세션 만료: 4시간 (일반 사용자보다 짧음)
- IP 제한 옵션 (페이즈 2)
- 로그인 실패 5회 시 30분 잠금
- 모든 액션 audit_logs 기록

### 3.2 키보드 단축키

| 키 | 액션 |
|----|------|
| `J` 또는 ← | 거절 |
| `K` 또는 → | 승인 |
| `L` 또는 ↑ | 대표 사진 지정 |
| `B` | 책갈피 (나중에 다시 보기) |
| `R` | 신고로 표시 |
| `C` | 카테고리 수정 모달 |
| `Space` | 큰 사진 보기 (라이트박스) |
| `?` | 단축키 도움말 |

빠르게 사용 가능. 시간당 1,000장+ 처리 가능.

### 3.3 거절 사유 입력

